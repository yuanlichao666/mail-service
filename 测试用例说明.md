# 测试用例说明

本文说明 `scripts/` 下 DuckMail 兼容 HTTP 与 SMTP 入站测试的**前置条件**、**测试目标**、**用例步骤**及**常用命令**。

## 线上环境说明（当前部署）

- **服务器公网 IP**：`47.253.177.201`（示例，以实际为准）。  
- **邮箱域名**（`config.yaml` 中 `domains[].domain`）：**`mail.goodfish.cloud`**（测试脚本默认使用配置中**第一个**域名，或通过 `MAIL_TEST_DOMAIN` 覆盖）。  
- **DuckMail HTTP API**：宿主机映射端口一般为 **3001**（`http://47.253.177.201:3001`），与 **80 端口**（如 Apache 欢迎页）无关。  
- **SMTP 入站**：Docker 将容器内 **2525** 映射到宿主机 **25**（标准 SMTP）。云安全组通常**只开放 25**，**不开放 2525**；从**公网**执行 `test:mail:remote` 时，脚本默认连接 **`47.253.177.201:25`**。若在服务器本机执行 `test:mail:local`，仍连 **`127.0.0.1:2525`**（或 `.env` 中 `HTTP_PORT` 同目录逻辑下的宿主机映射）。

## 前置条件

1. **配置文件**：项目根目录存在 `config/config.yaml`（可由 `config/config.example.yaml` 复制）。  
   - `duckmailApiKey`：非空；测线上时须与线上一致。  
   - `domains`：至少一条，且线上已配置相同域名（如 **`mail.goodfish.cloud`**）；测试使用其中**第一个** `domain` 作为邮箱后缀（或通过 `MAIL_TEST_DOMAIN` 指定）。  
   - 该域名在带 API Key 的 `GET /domains` 中须为 **`isVerified: true`**，否则 DuckMail API 用例会失败。

2. **依赖**：已执行 `npm install`（含开发依赖 `nodemailer`）。

3. **测试目标**：必须通过 `TEST_TARGET` 或下文 npm 脚本明确二选一：  
   - **local**：在你当前电脑上，访问本机运行的服务（默认 `127.0.0.1` + 端口）。  
   - **remote**：在你当前电脑上，访问已部署的公网服务（默认 HTTP `http://47.253.177.201:3001`；SMTP 默认 **`47.253.177.201:25`**，见上文端口说明）。

未设置 `TEST_TARGET` 时直接运行 `node scripts/*.cjs` 会报错，请使用下表中的 `npm run` 命令。

## 测试目标与环境变量

| 环境变量 | 说明 |
|----------|------|
| `TEST_TARGET` | 必填（若不用 npm 脚本）：`local` \| `remote`（`server`、`r` 等同 `remote`）。 |
| `MAIL_SERVICE_USE_LOCAL` | 设为 `1` 时等价于 `TEST_TARGET=local`。 |
| `MAIL_SERVICE_BASE` | 覆盖 HTTP Base URL（一般不必设，除非自定义网关）。 |
| `MAIL_REMOTE_BASE` | `remote` 时覆盖默认 HTTP 地址（默认 `http://47.253.177.201:3001`）。 |
| `MAIL_REMOTE_SMTP_HOST` | `remote` 时覆盖 SMTP 主机（默认 `47.253.177.201`）。 |
| `MAIL_REMOTE_SMTP_PORT` | `remote` 时覆盖默认 SMTP 端口（**不设则为 25**，与仅开放 25、不开放 2525 的安全组一致）。 |
| `MAIL_SERVICE_HOST_PORT` | `local` 时覆盖本机 HTTP 端口（未设时尝试读 `.env` 中 `HTTP_PORT`，再回退 `config.yaml` 的 `httpPort`）。 |
| `MAIL_SERVICE_API_KEY` | 覆盖 `config.yaml` 中的 `duckmailApiKey`。 |
| `MAIL_TEST_DOMAIN` | 覆盖用于生成测试邮箱的域名。 |
| `SMTP_TEST_HOST` / `SMTP_TEST_PORT` | 同时覆盖 local/remote 的 SMTP 连接目标（优先级最高，高级用法）。 |
| `CONFIG_PATH` | 指定配置文件路径。 |

## npm 命令一览

| 命令 | 含义 |
|------|------|
| `npm run test:mail:local` | 本机测本机：依次执行 DuckMail HTTP 全套 + SMTP 入站。 |
| `npm run test:mail:remote` | 本机测线上：同上，目标为默认公网地址（可配环境变量）。 |
| `npm run test:mail` | 与 `test:mail:local` 相同。 |
| `npm run test:duckmail:local` / `test:duckmail:remote` | 仅 DuckMail HTTP 测试。 |
| `npm run test:smtp:local` / `test:smtp:remote` | 仅 SMTP 入站测试。 |
| `npm run test:duckmail` | 同 `test:duckmail:local`。 |
| `npm run test:smtp` | 同 `test:smtp:local`。 |

**在本机跑 `local` 前**：需先启动服务，例如 `npm run build && npm run start:prod` 或 `npm run start:dev`。

**跑 `remote` 前**：确认本机网络可达公网 IP；云安全组放行 **HTTP 映射端口**（如 **3001**）及 **SMTP（通常 25）**；`duckmailApiKey` 与线上一致；本地 `config.yaml` 中测试域名与**线上一致**（如 `mail.goodfish.cloud`）。

## 套件一：DuckMail 兼容 HTTP API（`test-duckmail-api.cjs`）

| 序号 | 用例 | 步骤与期望 |
|------|------|------------|
| D1 | 健康检查 | `GET /health`，HTTP 2xx，JSON `status === "ok"`。 |
| D2 | 域名列表（带 Key） | `GET /domains`，Header `Authorization: Bearer <apiKey>`，返回 `hydra:member` 为数组，且包含配置中 `domain` 且 `isVerified` 为 true 的项。 |
| D3 | 创建账户 | `POST /accounts`，带 Key，body：`address`（随机本地部分@配置域名）、`password`、`expiresIn: 86400`；期望 HTTP 200 或 201。 |
| D4 | 换取邮箱 JWT | `POST /token`，body：`address`、`password`；期望 HTTP 200，JSON 含字符串 `token`。 |
| D5 | 邮件列表（空） | `GET /messages`，`Authorization: Bearer <邮箱 JWT>`；期望 HTTP 200，`hydra:member` 为数组（新建账户可为空列表）。 |
| D6 | 单封详情（不存在） | `GET /messages/00000000-0000-0000-0000-000000000000`，同上 JWT；期望 **HTTP 404**。 |

## 套件二：SMTP 入站（`test-smtp-inbound.cjs`）

| 序号 | 用例 | 步骤与期望 |
|------|------|------------|
| S1 | 创建账户 | 同 D3，使用新随机邮箱与密码。 |
| S2 | 换取 JWT | 同 D4。 |
| S3 | SMTP 投递 | 使用 nodemailer 连接当前目标 `smtpHost:smtpPort`，无认证，向该邮箱发送一封文本邮件（主题含时间戳，正文含 `123456`）。 |
| S4 | 轮询列表 | 最多约 10 秒内轮询 `GET /messages`，直到 `hydra:member` 非空，取第一条 `id`。 |
| S5 | 读信校验 | `GET /messages/:id`，期望主题与发信一致；`text` 或 `intro` 中含 `123456`。 |

## 常见问题

- **`必须指定测试目标`**：未设置 `TEST_TARGET` 时直接运行 `node scripts/...`，请改用 `npm run test:mail:local` 或 `test:mail:remote`。  
- **`DuckMail API Key 无效`（401）**：`remote` 时本地 `config.yaml` 的 `duckmailApiKey` 与服务器不一致，或设置 `MAIL_SERVICE_API_KEY`。  
- **`带 Key 的 /domains 应包含已验证域名`**：配置中该域名 `isVerified` 须为 `true`，且与线上一致。  
- **本机 HTTP 连错端口**：若服务跑在 Docker 且宿主机映射为 `3001`，可在项目根目录 `.env` 中配置 `HTTP_PORT=3001`，或设置 `MAIL_SERVICE_HOST_PORT=3001`。  
- **`remote` SMTP 超时**：公网通常连 **25**；若你在安全组中开放了 **2525** 并映射到容器，可设 `MAIL_REMOTE_SMTP_PORT=2525` 或 `SMTP_TEST_PORT=2525`。  
- **域名不一致**：本地测试用的 `domain` 须与服务器 `config.yaml` 中声明的域名一致（当前示例为 **`mail.goodfish.cloud`**）。
